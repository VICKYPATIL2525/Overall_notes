Objective

Understand what Nginx is, why it is needed, what you would have to do manually without it, and how to install and verify it on an EC2 server as the first step toward connecting it to a running application.

Overview

What is Nginx
Nginx (pronounced "engine-x") is a web server and reverse proxy. It sits in front of your application and handles all incoming traffic from the internet before passing it to your app. Think of it as a traffic controller — it decides where each request should go and manages the connection between the outside world and your internal services.
The Problem Without Nginx
When you run an app (like Streamlit) on an EC2 server, it starts on a specific port — for example http://your-ip:8501. Without Nginx, users must type that port in the URL, there is no way to use a clean domain name, no HTTPS, and every app port must be individually opened in the Security Group. Managing multiple apps on one server becomes messy fast.
What You Would Have to Do Manually
Without Nginx you would need to: open every app port (8501, 8502, etc.) in the EC2 Security Group individually; tell every user to add the port number to every URL; configure SSL certificates manually per app; and write custom scripts to route traffic between multiple apps. This is error-prone and does not scale.
How Nginx Solves This
Nginx acts as a reverse proxy — it receives all traffic on port 80 (HTTP) or 443 (HTTPS) and forwards it internally to your app on the correct port. Only one port needs to be open publicly. Users visit a clean URL with no port number. SSL is managed in one place. Multiple apps can run on the same server and Nginx routes each request to the right one.
End Result
After setup, traffic flows cleanly through Nginx before reaching your app: Internet → Port 80 → Nginx → Port 8501 → Your App. Users see a clean URL, your Security Group only exposes port 80/443, and all routing is managed in one Nginx config file.

Installation Steps

Step 01 Install Nginx

Installs the Nginx package from Ubuntu's default package repository. The -y flag auto-confirms any prompts.

sudo apt install nginx -y
Step 02 Start Nginx

Starts the Nginx service immediately. After this command, Nginx is running and listening on port 80.

sudo systemctl start nginx
Step 03 Enable Auto-Start on Reboot

Without this, Nginx only runs for the current session. This command registers it to start automatically every time the server boots up.

sudo systemctl enable nginx
Step 04 Check Status

Confirms Nginx started successfully. Look for Active: active (running) in the output.

sudo systemctl status nginx
Expected output — You should see Active: active (running). If it shows failed or inactive, refer to the Troubleshooting section below.
Step 05 Test in Browser

Opens the server's public IP on port 80 (the default HTTP port). If Nginx is running correctly, it serves its default welcome page.

http://your-elastic-ip
Expected — You should see the "Welcome to nginx!" page. This confirms Nginx is installed, running, and reachable from the internet.

Troubleshooting — If the Page Does Not Open

Check 01 EC2 Security Group — Port 80

The EC2 Security Group acts as a firewall. If port 80 is not in the inbound rules, traffic cannot reach the server at all — Nginx will never receive the request.

Fix — Go to EC2 → Security Groups → Inbound Rules → Add Rule: Type = HTTP, Port = 80, Source = 0.0.0.0/0.
Check 02 Server Firewall (UFW)

If UFW (Ubuntu's built-in firewall) is active, it may be blocking port 80 even if the Security Group allows it. This is less common but worth checking.

sudo ufw allow 'Nginx Full'

What Just Happened

At this point your server architecture looks like this:

Internet → Port 80 → Nginx → (default welcome page)

Nginx is running and handling traffic on port 80, but it is not yet connected to your Streamlit app. The welcome page is Nginx's built-in default. The next step is to update the Nginx configuration to forward traffic to your app running on port 8501.

Next Step — Connect Nginx to Streamlit

Once the "Welcome to nginx!" page loads successfully, you update the Nginx config to act as a reverse proxy — forwarding all incoming traffic to your Streamlit app running internally on port 8501. 

Internet → Port 80 → Nginx → Port 8501 → Streamlit App
Note — Config steps for this will be added in the next section once you confirm Nginx is working and you are ready to connect it to your app at app.yourdomain.com.

Additional Information

Reverse Proxy vs Web Server
Nginx can work in two modes. As a web server, it directly serves static files (HTML, CSS, images) to the browser. As a reverse proxy, it sits in front of another application and forwards requests to it. In most deployment setups — like Streamlit behind Nginx — it is acting as a reverse proxy. The app itself handles the response; Nginx just routes the traffic.
Why Port 80 and Not the App Port
Port 80 is the default HTTP port. Browsers automatically try port 80 when no port is specified in a URL. By having Nginx listen on port 80, users can visit http://your-ip without typing a port number. Nginx then forwards the request internally to your app on whatever port it uses (e.g., 8501), which never needs to be publicly exposed.
reload vs restart
systemctl restart nginx stops and restarts the process — there is a brief moment of downtime. systemctl reload nginx reloads the configuration file without stopping the process — existing connections are not interrupted. Always prefer reload when you only changed the config and the server is live.
nginx -t — Always Run Before Reloading
sudo nginx -t validates the Nginx configuration file for syntax errors before you apply it. If there is an error in your config and you reload without testing, Nginx may fail to start. Running nginx -t first is a safe habit — it tells you exactly which line has an error so you can fix it before anything breaks.
Where Nginx Config Files Live
The main config file is at /etc/nginx/nginx.conf. Individual site configs go in /etc/nginx/sites-available/ and are enabled by symlinking them into /etc/nginx/sites-enabled/. When you configure Nginx to forward traffic to your app, you will create a file in sites-available/ and enable it from there.
